# 通用方法整理与介绍
    - Foundationpose 提出统一的 model-based 和 model-free 6D 位姿估计框架，可以在给定 CAD 或少量参考图像的情况下迁移到新目标。
    - SAM-6D 使用 SAM 做零样本 6D 位姿估计，面向 RGB-D 输入中的新物体检测和位姿估计。
    - Any6D 进一步把问题推进到只需要单张 RGB-D anchor image，就估计未知物体的 6D 位姿和尺寸。

# 航天器专用的“通用位姿估计”问题
### 定义类似NOCS规范坐标空间表示
    - Spacecraft Canonical Coordinate Space，SCCS
    - 结构部件分割 + 规范坐标 + 不确定性 + 尺度参数 + 6D/9D 位姿
### 无精确 CAD 或少样本参考
    - Known-CAD setting：给定目标 CAD，作为基础版本。
    - Few-shot reference setting：给定几张参考图像或少量视角。
    - CAD-free / weak-CAD setting：没有精确 CAD，只知道这是航天器类别，或者只有粗糙几何模型。
```
第一篇：已知航天器下的轻量化高精度位姿估计，也就是你现在 ConvMamSPN + UGGP-ST 的延续。

第二篇：多航天器/跨实例的结构感知 6D 位姿估计。

第三篇：无精确 CAD 或少样本参考的通用航天器 6D/9D 位姿估计。

第四篇：事件相机、RGB-D 或时序融合下的鲁棒位姿估计。

第五篇：嵌入式部署和视觉伺服验证。
```

### 数据集

| 数据来源             | 适合用途                               | 局限                 |
| ---------------- | ---------------------------------- | ------------------ |
| SPEED / SPEED+   | 单实例、合成到真实、光照域适应                    | 目标单一，难支撑通用性        |
| URSO             | 多航天器合成数据，如 Soyuz、Dragon            | 规模有限，主要是合成域        |
| SPARK 2022       | 多类空间目标检测/轨迹相关任务，含多种目标              | 不一定直接适合 6D 通用位姿主任务 |
| SpaceSense-Bench | 多模态、多卫星、多部件语义、RGB/Depth/LiDAR/6DoF | 新数据集，需检查标注尺度和任务定义  |
| SPADES           | 事件相机航天器位姿估计                        | 更适合事件视觉支线          |


### 学习

1.航天器结构规范空间 SCCS。借鉴 NOCS，但不能照搬。NOCS 的核心思想是把同一类别下不同实例映射到统一规范坐标空间，用于类别级 6D 位姿和尺寸估计。NOCS 原始问题设定就是在没有精确 CAD 的情况下，对未见过实例进行 6D 位姿和尺寸估计。
